# DreamWeaver 🌙

DreamWeaver არის სიზმრების დღიური (dream journal) — აპლიკაცია, რომელშიც მომხმარებელს შეუძლია ჩაიწეროს და შეინახოს საკუთარი სიზმრები, მონიშნოს ისინი როგორც „ღამისეული" (lucid), დაალაგოს და გაფილტროს ისინი, და რაც მთავარია — **შეანჯღრიოს ტელეფონი**, რომ მიიღოს შემთხვევითი შთაგონების ფრაზა სიზმრის აღწერისას, თუ ვერ იხსენებს დეტალებს.

## ფუნქციონალი

- **სიზმრების სია** — მთავარ ეკრანზე `LazyColumn`-ში ნაჩვენებია ყველა ჩაწერილი სიზმარი, თარიღით, განწყობის ემოჯით და lucid ნიშნით.
- **მენიუ** — `TopAppBar`-ზე განთავსებული `DropdownMenu`-ს მეშვეობით შესაძლებელია:
  - მხოლოდ ღამისეული (lucid) სიზმრების ჩვენება;
  - სორტირება უახლესით/უძველესით;
  - დღის შთაგონების განახლება.
- **ახალი სიზმრის დამატება** — ქვედა ფურცელზე (`ModalBottomSheet`) მომხმარებელი წერს სათაურს, აღწერას, ირჩევს განწყობას (ემოჯი-ჩიფები) და უთითებს იყო თუ არა lucid.
- **შეანჯღრიე შთაგონებისთვის (Shake-to-Inspire)** — პროექტის ახალი, აქამდე გამოუყენებელი ფუნქციონალია მოწყობილობის **აქსელერომეტრის (accelerometer)** გამოყენება. სანამ „ახალი სიზმრის" ფურცელი ღიაა, ტელეფონის შენჯღრევისას აპლიკაცია აღწერის ველს ავტომატურად ამატებს შემთხვევით წერის ბიძგ-კითხვას (prompt), რათა დაეხმაროს მომხმარებელს დავიწყებული დეტალების გახსენებაში.
- **დღის შთაგონება** — მთავარი ეკრანის თავზე ბანერი აჩვენებს დღის შემთხვევით „დაკვირვებას/რჩევას", რომელიც წამოღებულია დისტანციური REST API-დან.

## არქიტექტურა — MVVM

პროექტი აგებულია **MVVM**-ზე, მთლიანად Jetpack Compose-ით (`findViewById` არ გამოიყენება):

```
ui/
 ├─ screens/DreamListScreen.kt   → View (Composable ფუნქციები: სია, მენიუ, ქვედა ფურცელი)
 ├─ DreamViewModel.kt            → ViewModel (StateFlow<DreamScreenState>)
 ├─ DreamViewModelFactory.kt     → ViewModel-ის მანუალური დამოკიდებულებების ინექცია
 └─ state/DreamScreenState.kt    → UI მდგომარეობის immutable მოდელი (ფილტრი/სორტი)

data/
 ├─ model/Dream.kt               → დომენის მოდელი + განწყობის emoji მეპინგი
 ├─ remote/                      → Retrofit (AdviceApiService, DTO-ები, RetrofitInstance)
 ├─ local/                       → Room (DreamEntity, DreamDao, DreamDatabase)
 └─ repository/DreamRepository.kt→ Model (აერთიანებს Room-ს და Retrofit-ს)

util/
 ├─ Resource.kt                  → sealed class ქსელური მდგომარეობისთვის
 ├─ ShakeDetector.kt             → აქსელერომეტრზე დაფუძნებული შენჯღრევის დეტექტორი (ახალი ფუნქციონალი)
 └─ DreamPrompts.kt              → ლოკალური წერის ბიძგ-კითხვების ბანკი
```

**ნაკადი:** `View` ისმენს `ViewModel`-ის `StateFlow`-ს → `ViewModel` სიზმრებს იღებს `Repository`-დან → `Repository` აერთიანებს ორ წყაროს: `Room`-ში შენახულ სიზმრების დღიურს და `Retrofit`-ით წამოღებულ დღის შთაგონებას. View-ს არასდროს აქვს პირდაპირი წვდომა ბაზასთან ან ქსელთან.

## ბაზასთან კავშირი

| ტექნოლოგია | დანიშნულება |
|---|---|
| **Room** | მომხმარებლის ყველა სიზმრის ჩანაწერის ლოკალური, მუდმივი შენახვა (`dreams` ცხრილი) — მთავარი მონაცემთა წყარო |
| **Retrofit + Gson** | დღის შემთხვევითი შთაგონების/რჩევის წამოღება საჯარო, უფასო API-დან [`api.adviceslip.com`](https://api.adviceslip.com) (გასაღები არ სჭირდება) |

## ტექნიკური დეტალები

- **ენა:** Kotlin
- **UI:** Jetpack Compose + Material 3 (`findViewById` არ გამოიყენება)
- **არქიტექტურა:** MVVM, `StateFlow`, Coroutines
- **ქსელი:** Retrofit2, OkHttp logging interceptor, Gson converter
- **ბაზა:** Room (KSP ანოტაციების დამუშავებისთვის)
- **სენსორი:** Android `SensorManager` + `TYPE_ACCELEROMETER` (შენჯღრევის ამოცნობა)
- **მინიმალური SDK:** 24 (Android 7.0)
- **სამიზნე/Compile SDK:** 34

## პროექტის გაშვება

1. გახსენით root საქაღალდე Android Studio-ში (Hedgehog ან უფრო ახალი).
2. დაელოდეთ Gradle sync-ის დასრულებას.
3. გაუშვით აპლიკაცია — ემულატორზე ტესტირებისთვის შენჯღრევის სიმულაცია შესაძლებელია Extended Controls → Virtual Sensors → Accelerometer-იდან; რეალურ მოწყობილობაზე უბრალოდ შეანჯღრიეთ ფიზიკურად.
4. დააჭირეთ „+" ღილაკს ახალი სიზმრის დასამატებლად და სცადეთ შენჯღრევა შთაგონებისთვის.

## შესაძლო გაუმჯობესებები მომავალში

- Firebase Realtime Database-ის დამატება სიზმრების ღრუბელში სინქრონიზაციისთვის რამდენიმე მოწყობილობას შორის.
- სიზმრების სტატისტიკა (ყველაზე ხშირი განწყობა, lucid სიზმრების პროცენტი თვის მიხედვით).
- შეტყობინებები (WorkManager) ღამის დღიურის შესავსებად შეხსენებისთვის გაღვიძებისას.
